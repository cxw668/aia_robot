## 二、[01]-product-产品方案.md（核心内容）

### 1) 产品目标

构建一个“轻量级 AIA 企业机器人”，支持：

- 企业知识问答（优先基于 `@aia_data/`）
- 结构化内容检索（产品、表单、分公司、服务菜单等）
- 可扩展文件下载入库（后续通过链接自动拉取并更新知识库）
- Agent 任务编排能力（未来扩展）

### 2) 核心能力边界（MVP）

- 多轮问答
- 基于向量检索的RAG回答
- 来源引用（返回命中知识片段/文档名）
- 后台可触发知识入库
- **服务需要创建但是暂无使用**
  - 复杂流程审批
  - 多Agent协同
  - 语音/视频能力

### 3) 目标用户

- 内部客服/运营/销售支持人员
- 需要快速查询产品、表单、分公司信息的员工

## 三、[02]-architecture-技术架构.md

### 1) 总体分层

1. **前端层（React）**
  - 聊天交互、会话历史、引用来源展示、反馈按钮
2. **服务层（Python）**
  - API网关、会话管理、RAG编排、Agent调度、权限控制
3. **知识层（向量数据库 + 元数据存储）**
  - 向量索引、文本切片、文档版本、标签过滤
4. **模型层（硅基流动免费模型API）**
  - Embedding模型 + Chat模型
5. **数据接入层**
  - 本地目录接入（`@aia_data/`）
  - URL下载接入（后续）

### 2) 核心链路（RAG）

用户问题 → Query改写（可选）→ 向量检索（TopK）→ 重排（可选）→ 拼接上下文 → LLM生成 → 返回答案+引用

### 3) 推荐技术选型（轻量优先）

- 前端：React + Vite + Zustand+ Material UI
- 后端：Python + FastAPI
- 向量库：Qdrant（轻量、易部署）  
- 元数据存储：PostgreSQL 
- 对象存储（可选）：MinIO（用于原始文件与解析产物）
- 任务队列（后续）：Celery/RQ + Redis

## 四、[03]-data-知识库与向量化方案.md（核心内容）

### 1) 数据分层

- **L0 原始层**：`@aia_data/` 原始 JSON
- **L1 标准层**：统一字段结构（title/content/type/source/version）
- **L2 切片层**：chunk 后文本
- **L3 索引层**：向量 + 元数据索引

### 2) 切片策略（JSON场景）

- 以业务对象为主切（如“产品条目”“分公司条目”）
- 每片建议 300–800 中文字
- 保留层级路径（如 `产品>保障责任>条款`）作为 metadata
- metadata 必含：`doc_id` `source_file` `category` `updated_at` `version`

### 3) 更新策略

- 首次全量入库
- 后续增量更新（按文件hash/version）
- 删除策略：软删除 + 定时压实

### 4) URL下载入库扩展

- 维护“链接任务表”（URL、类型、抓取时间、状态）
- 下载→解析→标准化→切片→向量化→索引
- 失败重试与任务可观测（状态机）

## 五、[04]-agent-agent与模型调用方案.md（核心内容）

### 1) Agent职责（单Agent起步）

- 意图识别（问答/查表单/查分公司）
- 检索策略选择（按category过滤）
- 调用RAG并组织答案
- 输出结构化结果（answer + citations + confidence）

### 2) 模型调用策略（硅基流动免费API）

- **双模型模式**
  - Embedding模型：仅用于向量化和query向量化
  - Chat模型：用于最终回答
- **鲁棒策略**
  - 超时控制
  - 重试（指数退避）
  - 限流与配额统计
  - 降级（无LLM时返回检索结果摘要）

### 3) Prompt原则

- 严格“仅依据检索上下文回答”
- 无依据时明确回复“未在知识库中找到”
- 强制附带来源引用（文件名/条目ID）

## 六、[05]-frontend-react前端方案.md（核心内容）

### 1) 页面最小集合

- 登录页（可选，内部系统可走SSO）
- 聊天页（主页面）
- 知识管理页（入库任务、状态、日志）

### 2) 聊天页核心模块

- 会话区（多轮）
- 回答区（支持 markdown）
- 引用区（显示命中文档与片段）
- 反馈区（有帮助/无帮助）

### 3) 前后端接口契约（核心）

- `POST /chat`：提问并返回答案、引用、置信度
- `POST /kb/ingest`：触发目录或URL入库
- `GET /kb/jobs`：查询入库任务状态
- `GET /health`：健康检查

## 七、[06]-backend-python后端方案.md（核心内容）

### 1) 服务模块划分

- `api`：HTTP接口
- `orchestrator`：RAG/Agent流程编排
- `retriever`：向量检索与过滤
- `ingestion`：文件解析与入库
- `model_client`：硅基流动API封装
- `storage`：向量库与元数据访问

### 2) 核心数据对象

- `Document`（文档）
- `Chunk`（切片）
- `Citation`（引用）
- `ChatSession`（会话）
- `IngestJob`（入库任务）
